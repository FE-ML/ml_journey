# Дневник тренировок

## Стек
- Python 3.12
- Numpy
- uv
- Git
- VS Code

## Запуск проекта
### Создание виртуального окружения
```bash
uv venv
```
### Активация окружения
```bash
.venv\Scripts\Activate.ps1
```
### Запуск проекта
```bash
uv run hello.py
```

## Краткая информация про то, как устроено окружение:
1. Интерпретатор Python - приложение, которое запускает файлы формата py. 
    Важны версии интерпретатора, потому что от этого зависит, какие библиотеки будут доступны для работы с данным интерпретатором. Сейчас самыми актуальными версиями являются 3.11 и 3.12.
2. Виртуальное окружение - отдельная папка со своей копией python, версиями библиотек и настройками.
    Пакеты нельзя ставить глобально, потому что проекты могут использовать разные версии библиотек, а использование глобавльного пакета их сломает.
3. pyenv - инструмент для управления версиями интерпретатора Python на компьютере. Позволяет использовать разные версии интерпретатора в разных проектах.
4. uv - современный инструмент для управления зависимостями и окружениями. Заменяет испльзование pip, venv, pip-tools.
5. Git - система контроля версий.
6. GitHub - веб-сервис для хранения git репозиториев.
7. Репоситорий - папка проекта, в которой git отслеживает изменения файлов.
8. Комит - снимок состояния проекта.
9. Ветка - независимая линия разработки, которая позволяет вести независимую работу в коде, не трогая ее основную версию.
10. Remote - удаленный репозиторий, который хранится на сервере.
11. Push - команда, позволяющая загрузить локальные изменения в удаленный репозиторий.
12. Pull - команда, позволяющая загрузить даныные из удаленного репозитория в локальное хранилище.
